Key Concepts Learned

Defensive security focuses on protecting systems, detecting threats, and responding to incidents. While offensive security tries to break into systems, defensive security works to keep attackers out and minimise damage when breaches occur.


Defensive Security

Defensive security is the practice of protecting systems, networks, and data from attacks.

Core goals of defensive security include:


Preventing unauthorised access
Detecting threats and suspicious activity
Responding to incidents quickly and effectively
Recovering systems after an attack


Defenders work to maintain the confidentiality, integrity, and availability outlined in the CIA Triad.


The Security Operations Centre (SOC)

A Security Operations Centre, or SOC, is a team of security professionals responsible for monitoring and defending an organisation's systems around the clock.

Key responsibilities of a SOC include:


Monitoring systems and networks for signs of attack
Analysing alerts and triaging potential threats
Investigating security incidents
Coordinating incident response
Reporting on security posture and trends


SOC analysts are often the first line of defence when an attack is detected.


Threat Monitoring and Detection

Defenders use a range of tools and techniques to identify threats before they cause serious damage.

Methods of threat monitoring include:


Monitoring network traffic for unusual patterns
Reviewing system and application logs
Setting up alerts for known attack signatures
Using threat intelligence to stay informed about emerging threats


Early detection is critical because the sooner a threat is identified, the less damage it can cause.


Incident Response

When a security incident occurs, defenders follow a structured process to contain and resolve it.

A typical incident response process includes:


Identification — detecting and confirming that an incident has occurred
Containment — limiting the spread or impact of the incident
Eradication — removing the threat from the affected systems
Recovery — restoring systems to normal operation
Lessons learned — reviewing the incident to improve future defences


Effective incident response minimises downtime, data loss, and reputational damage.


Firewalls, IDS, and IPS

Defenders use several key technologies to protect networks and systems.

Firewall:


Filters incoming and outgoing network traffic based on rules
Blocks unauthorised connections
Acts as a barrier between trusted and untrusted networks


Intrusion Detection System (IDS):


Monitors network traffic for suspicious activity
Generates alerts when potential threats are detected
Does not block traffic — only detects and alerts


Intrusion Prevention System (IPS):


Similar to an IDS but takes active steps to block threats
Can automatically drop malicious traffic
Provides a more proactive layer of protection


These tools work together to monitor, detect, and prevent attacks at the network level.


Log Analysis

Logs are records of events that occur across systems, applications, and networks. Analysing logs is one of the most important skills in defensive security.

Logs can reveal:


Failed and successful login attempts
Unusual access patterns
System errors and crashes
Evidence of malware or unauthorised tools
Data exfiltration attempts


Security professionals use tools such as SIEM (Security Information and Event Management) platforms to collect, correlate, and analyse large volumes of log data efficiently.


Why This Matters

Defensive security is essential for protecting individuals, organisations, and critical infrastructure.

Defensive skills are applied in roles such as:


SOC analyst
Incident responder
Threat intelligence analyst
Security engineer
Digital forensics investigator


Understanding how defenders operate provides a complete picture of the cyber security landscape alongside offensive knowledge.
