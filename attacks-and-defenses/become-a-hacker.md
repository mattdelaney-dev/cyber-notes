# Become a Hacker

## Overview

This room introduced offensive security and the role of ethical hackers in cyber security.

Topics explored included:
- What offensive security is
- The role of a penetration tester
- Reconnaissance techniques
- Exploiting vulnerabilities
- Common offensive tools
- Ethical hacking principles

## Key Concepts Learned

Offensive security involves thinking and acting like an attacker in order to identify and fix vulnerabilities before malicious hackers can exploit them. Ethical hackers are authorised to carry out attacks on systems to improve their security.

---

## Offensive Security

Offensive security is the practice of proactively attacking systems, networks, and applications to find weaknesses.

Unlike defensive security, offensive security focuses on:
- Finding vulnerabilities before attackers do
- Simulating real-world attacks in a controlled way
- Providing actionable findings to improve security

The goal is not to cause harm but to expose weaknesses so they can be fixed.

---

## Penetration Testing

Penetration testing (pentesting) is a structured approach to testing a system's security by attempting to break into it with permission.

A penetration test typically follows these stages:
- Planning — defining scope and rules of engagement
- Reconnaissance — gathering information about the target
- Scanning — identifying open ports, services, and vulnerabilities
- Exploitation — attempting to gain unauthorised access
- Reporting — documenting findings and recommendations

Penetration testers help organisations understand their real-world exposure to attack.

---

## Reconnaissance

Reconnaissance is the first step in any offensive engagement. It involves gathering as much information about a target as possible before attempting an attack.

Types of reconnaissance include:
- Passive reconnaissance — collecting information without directly interacting with the target (e.g. searching public records, websites, social media)
- Active reconnaissance — directly interacting with the target to gather information (e.g. scanning ports, probing services)

Information gathered during reconnaissance can include:
- IP addresses and domain names
- Open ports and running services
- Employee names and email formats
- Technology stack and software versions

---

## Exploiting Vulnerabilities

Once vulnerabilities are identified, an attacker or ethical hacker may attempt to exploit them to gain access to a system.

Common vulnerability types include:
- Unpatched software
- Misconfigured services
- Weak or default credentials
- Injection flaws (e.g. SQL injection)
- Broken authentication

Exploiting a vulnerability can allow an attacker to:
- Gain unauthorised access to a system
- Escalate privileges
- Move laterally through a network
- Exfiltrate sensitive data

---

## Common Offensive Tools

Ethical hackers use a range of tools to carry out their work, including:
- Nmap — for network and port scanning
- Metasploit — for developing and launching exploits
- Burp Suite — for testing web application security
- Gobuster — for directory and file enumeration
- Hydra — for brute-forcing login credentials

These tools are used by both attackers and defenders, which is why understanding them is important for both roles.

---

## Ethical Hacking Principles

Ethical hacking is only legal and legitimate when carried out within a defined scope with proper authorisation.

Key principles include:
- Always obtain written permission before testing
- Stay within the agreed scope
- Do not cause unintended damage or disruption
- Report all findings honestly
- Protect any sensitive data encountered during testing

Ethical hackers operate under the same legal framework as any professional service. Hacking without permission is a criminal offence.

---

## Why This Matters

Understanding offensive techniques is valuable for both attackers and defenders.

Offensive security skills are used in:
- Penetration testing and red teaming
- Bug bounty programmes
- Vulnerability research
- Security audits
- Capture the Flag (CTF) competitions

Thinking offensively helps security professionals understand how attackers operate and how to better defend against them.
